Media Query
Media queries is a CSS3 module allowing content rendering to adapt to conditions such as screen resolution (e.g. smartphone screen vs. computer screen).
@media screen and (min-width: 480px) {
  body {
    background-color: lightgreen;
  }
}
Open 
@media screen and (max-width: 900px) and (min-width: 600px) {
  div.example {
    font-size: 50px;
    padding: 50px;
    border: 8px solid black;
    background: yellow;
  }
}
Open
Options list:
  • Min Width
  • Max Width
  • Min Height
  • Max Height
  • Orientation (landscape|portrait)

What is the problem of putting the media queries in every component?
Solution?
Create media queries variables.
$break1: "(max-width: 767px)";
$break2: "(min-width: 768px) and (max-width: 1185px)";
$break3: "(min-width: 1186px)";

Open or Closed Breakpoints?

Open Breakpoints
@media (min-width: 500px) {
  /*our styles*/
}
Closed Breakpoints
@media (min-width: 500px) and (max-width: 1000px)  {
  /*our styles*/
}

Open or Closed Breakpoints?

Don't use Open Breakpoints
Use only Closed Breakpoints!
unless it is the last breakpoint
Use the the same styles in more than 1 breakpoint
.other-component {
  /*some styles*/
  /*tablet and desktop*/
  @media #{$break2}, #{$break3} {
    color: green;
  }
  /*tablet*/
  @media #{$break2} {
    font-size: 22px;
  }
  /*desktop*/
  @media #{$break3} {
    font-size: 18px;
  }
}

“Desktop First” or “Mobile First”?

.other-component {
  /*common style only!*/
  
  /*tablet and desktop*/
  @media #{$break2}, #{$break3} {...}
  
  /*only mobile*/
  @media #{$break1} {...}
  
  /*only tablet*/
  @media #{$break2} {...}
  
  /*only desktop*/
  @media #{$break3} {...}
}

How many breakpoints?

  • How many breakpoints the web designer wants.
  • 2–4 breakpoints, Will be sufficient in most cases.
