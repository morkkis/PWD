Web Application

“The use case of an application is always to DO something with it”.

  • Rich/interactive user interface
  • Action-oriented rather than information oriented.
  • Creation versus Consumption
  • Not relying heavily on (or hiding when possible) the browser chrome (back button, reload button, address bar)
Why Web Application?
  • Better UX
  • Ease of development
  • Ease of Maintenance
  • Cross-platform
  • Mobility
  • No need for “Market Place”

CSS Display
  • display: block
  • display: inline
  • display: inline-block
Display: block
  • Take full-width (100% width) by default
  • Each gets displayed in a new line
  • Width & height properties can be set
  • Can contain other block or inline elements
Display: inline
  • Take only as much space as they need
  • Displayed side by side
  • Don’t accept width or height properties, and top-bottom margin
  • Can be a parent of other inline elements
