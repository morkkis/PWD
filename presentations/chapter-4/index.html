Component Precipices
  • Focus One SectionOne component should focus on one section of the UI
  • Access own HTMLOne component shouldn’t access another’s html directly
  • Encapsulated AssetsThe component encapsulates its HTML, CSS and JS
  • Comunication with OthersEach component has some way to interact with others
Component Advantages
  • Single Responsibility PrincipleAligns with SRP (every module, class, or function should have responsibility over a single part of the functionality)
  • Separation of ConcernsEach component leaving by itself and not effected from other components
  • Independent & ReusableLet you split the UI into independent, reusable pieces, and think about each piece in isolation
Due to these advantages every major framework or library like React, Vue, Angular uses this.

Redux - Three Principles:

  • Single source of truthThe state of your whole application is stored in an object tree within a single store.
  • State is read-onlyWe are changing via action (The only way to change the state is to emit an action, an object describing what happened).
  • Changes are made with pure functionsNot changing the current state object, we will return a new state object. (Immutability)
CSS Issues
  • CSS is a global scope
  • Specificity War, Inheritance issues
  • Name clashing
First Solutions
  • No more ID's in selectors
  • No "brute-force" values (as that z-index)
  • No more !important or exotic CSS properties
  • No strict bindings between CSS and the DOM structure
  • Keep the specifity as low as possible, ideally 0.1.0
  • Use namespaces for your classes.
